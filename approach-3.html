Approach 3: Deep Learning based approach


Deep Learning approach has proven its significance from past 2 decades that it can perform very well without disclosing any complexities and not requiring any special kind of internmediate training phases which need any human interations.
So, we went further with implementing DL based approaches. For that, we have divided out dataset in such manner:
  1. Training Dataset - 74 videos
  2. validation Dataset - 15 videos
  3. Testing Dataset - 15 videos

APPROACH 3.1

Approach 3.1 consists of MobileNetv2 model - which is used widely to run on small and low configuration based devices such as mobile and in this project - on Nvidia Jetson Nano (More about it on Deployment page).


So in this pre-trained model, to specify the custom input image shape [270,480] for this project, We have added new additional layers to build a custom MobileNetV2 architecture and train it on our training video dataset, in addition to applying testing and validation sets in model architecture training process. The python script for this architecture is available here.

APPROACH 3.2

Due to issues we were facing with custom MobileNetv2 model architecture and such standard architectures are built and used for general purpose applications, we moved to next DL approach that is to create our own custom model from scratch for application-specific purpose in the project.

APPROACH 3.3

Our model with dilated convolutions
We needed a much faster and light-weight model so that it can run on small computing devices like Jetson Nano on a realtime-basis on live video stream.
In this approach, we have applied a concept known as Dilated Convolution in our custom model from approach 3.2. You can get more idea about this from the reference about dilated convolutions on erogol.com.
By using dilated convolution in the model architecture for our advantange, we are able to achieve higher throughput (number of predictions of video frames per second) without affecting the accuracy of prediction model.
